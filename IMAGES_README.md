# Добавление собственных изображений товаров

## Шаг 1: Подготовка изображений

1. **Создайте папку для изображений** (уже создана):
   ```
   public/images/
   ```

2. **Добавьте свои фото товаров** в папку `public/images/`:
   - Форматы: JPG, PNG, WebP
   - Размер: рекомендуется 500x500px или больше (квадратные)
   - Названия: используйте понятные имена файлов

   Примеры названий файлов:
   ```
   iphone-16-pro-max.jpg
   samsung-s25-ultra.jpg
   macbook-air-m3.jpg
   sony-wh-1000xm5.jpg
   ```

## Шаг 2: Настройка маппинга

1. **Откройте файл** `update-custom-images.js`

2. **Найдите объект** `imageMapping` и добавьте соответствия:
   ```javascript
   const imageMapping = {
     // Ваши товары
     'iPhone 16 Pro Max': 'iphone-16-pro-max.jpg',
     'Samsung Galaxy S25 Ultra': 'samsung-s25-ultra.jpg',
     'MacBook Air M3': 'macbook-air-m3.jpg',
     // Добавьте свои...
   };
   ```

3. **Или используйте пример** из `image-mapping-example.js` - скопируйте нужные строки

## Шаг 3: Обновление базы данных

1. **Запустите скрипт**:
   ```bash
   node update-custom-images.js
   ```

2. **Проверьте вывод** - скрипт покажет, сколько изображений обновлено

## Шаг 4: Просмотр результатов

1. **Перезагрузите браузер** или очистите кэш
2. **Откройте каталог товаров** - изображения должны появиться в карточках

## Советы

- **Качество изображений**: Используйте четкие фото товаров с хорошим освещением
- **Размеры**: Для карточек подойдут изображения 300x300px минимум
- **Формат**: JPG для фото, PNG для изображений с прозрачностью
- **Оптимизация**: Сжимайте изображения для быстрой загрузки

## Альтернативный способ

Если хотите обновить изображение для конкретного товара:

```javascript
// В update-custom-images.js добавьте:
'iPhone 16 Pro Max': 'my-custom-iphone-photo.jpg',
```

Затем положите файл `my-custom-iphone-photo.jpg` в папку `public/images/` и запустите скрипт.

## Устранение проблем

- **Изображение не показывается**: Проверьте путь `/images/filename.jpg` в браузере
- **Скрипт не работает**: Убедитесь, что файл находится в корне проекта
- **База данных**: Проверьте, что сервер запущен и база доступна</content>
